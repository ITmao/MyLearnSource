calendargroup日期组组件

需要成对录入日期时使用,包装日期第三方插件(lhgcalendar)的组件

依赖组件

  • lhgcalendar

适用范围

约束

  1. minDate,maxDate,format日期格式支持：yyyyMMdd、yyyy-MM-dd、yyyy/MM/dd
  2. 第二个框的录入日期必须大于第一个框的录入的日期

注意事项

  1. 需要使用panel组件做为容器
  2. 请使用 name1获取组件的Horn对象
  3. 当日期输入框中存在合法日期时，日期列表会默认选中该日期，否则选中当天日期（在最小-最大日期范围时）

组件的配置选项如:日期格式(注意！如果填写了config，就不能再调用date验证规则，例如：#calendar({"label":"日期校验", "name": "test5", "value":"中文", "check": "required;date;", "config":"{format:'yyyy-MM-dd'}"})；date和config只能选择一个)
minDate和maxDate的配置必须为类似2012-12-09的格式;另外由于第三方组件对中文格式化的支持有限config中format不能配置有中文分隔符如yyyy年MM月
用户配置的format会自动的识别，如果配置到h就会显示到选择小时的界面，如果配置到m就会显示到选择月份的界面为止
另外，用户配置的format和minDate/maxDate的格式必须一致

		        					
		        						示例：
		        						
		        							
你也可以指定一个符合 ISO-8601 格式的日期时间：
