Horn UI基础对象

Horn的js对象，提供获取UI组件的功能

适用范围

Horn本身并不提供宏组件，只是javascript对象，但提供的获取UI组件的方法。

注意事项

  1. 使用Horn.ready注册初始函数，需要注意执行顺序：
    head资源引入的顺序，如果是在components的js加载之前加载，则之前的jquery.ready会先被执行；
    在components的js加载之后的脚本以及代码中的片段代码遵守以下规则：
    Horn.ready的执行顺序优先于jquery.ready
    多个Horn.ready按照代码顺序执行
  2. 所有组件的名称（name）不能重名，否则使用名称取组件时只能获取到最后一个
  3. 所有组件的ID（id）不能重名，否则使用id 取组件时只能获取到第一个

方法

名称 参数 描述
getComp (name)

静态方法 工具组件的名称获取组件，此方法为静态方法

参数:
  • String:name 组件名称所有组件的名称不能重复，否则只能取到最后一个
    • 返回值:
  • Object: comp 组件对象
    • 示例： 
    Horn.getComp("compName")
    ready (fun)

    静态方法 在页面渲染完成后执行的方法，可注册多次，使用先进先出的执行顺序。
    执行顺序如下：
    head资源引入的顺序，如果是在components的js加载之前加载，则之前的jquery.ready会先被执行；
    在components的js加载之后的脚本以及代码中的片段代码遵守以下规则：
    Horn.ready的执行顺序优先于jquery.ready
    多个Horn.ready按照代码顺序执行

    参数:
  • Function:fun 函数
    • 返回值:
  • void:
    • 示例： 
    Horn.ready(function(){
  //TODO
});
    getCompById (id)

    静态方法 根据组件id获取组件，此方法为静态方法

    参数:
  • String:id 所有组件的ID不能重复，否则只能取到第一个
    • 返回值:
  • :
  • Object: comp 组件对象
    • 示例： 
    Horn.getCompById("compId");
    getDict (dictName, keys)

    根据字典名及key获取value（建议使用Horn.Util.getDicts）

    参数:
  • string:dictName 字典名
  • boolean:keys 字典key，可以为单个或多个，多个key以逗号分隔
    • 返回值:
  • String: values 字典value，单个或多个，多个value以逗号分隔（当key未定义时返回整个字典）

    • Defined in: horn.Base.js
